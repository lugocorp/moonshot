Moonshot brings static typing to lua

Lua Redefined

Moonshot is a compiled, optionally type-checked extension language that compiles to Lua. It allows programmers to refactor much larger codebases without experiencing type discrepancies at runtime. Programmers can define a variable with any of Lua's primitive types, or use typedefs to represent composite types. Moonshot even provides an inheritance-based class system, so you can defined interfaces and classes to your heart's content. Never deal with large, hard-to-maintain untyped codebases in Lua again. With Moonshot you can access the power of the entire Lua ecosystem...but through a compiled language frontend!

Moonshot was designed to be as similar to Lua as possible. The new language features all follow a Lua-like syntax, introducing several new keywords such as class, interface, typedef and where. This design choice was made to minimize the learning curve that sits between Lua and Moonshot. We hope that you'll enjoy using Moonshot for all your larger Lua-based projects! If you're ready to get started then check out down below.

Features

Optional Type System

Moonshot is like the TypeScript of Lua. It allows you to define variable that must hold a particular type, or you can use untyped varibles by default. It's up to you what structure you'd like to encode into your projects. Moonshot will simply enforce your own rules at compile-time.

Inheritance-Based OOP

Moonshot brings inheritance-based object-oriented programming to the Lua programming language. Create interfaces and classes to your heart's content. This allows you to reuse code and enforce access policies, all while leveraging Lua's rich ecosystem.

Interoperability

Lua is a very lightweight, embeddable language. You can install and run it on just about any system you want. Moonshot was designed as an extension over Lua's grammar, so you just write pure Lua with it if you wanted to. Moonshot also compiles into basic Lua code, without adding any additional runtime type checks. This means that using Moonshot will have no effect on your code's runtime performance.

Get started

It's easy to get started with Moonshot. Go download the project from the GitHub page. Then, you can build the project using the directions that you'll find there, or on the install page. You'll want to make sure you have the make tool on your system.

Once you've successfully built the project, you can start using the command line tool. It's a simple interface that takes in a Moonshot or Lua source file and outputs some Lua code. You can check out our reference section to learn how to write Moonshot code.
